# 어드민 대시보드 변경 명세

> 기준 시점: 2026-06-04 커밋 `5b1d54f` ~ `98d35c4` (dev 브랜치)  
> 작성일: 2026-06-04  
> 대상 파일: `admin-dashboard.html`

---

## 1. 개요

어드민 대시보드의 플레이리스트 생성 흐름과 음원 데이터 관리 UX를 전면 개편하였습니다. 주요 변경 방향은 다음과 같습니다.

- **무드 매핑**: 별도 사이드바 메뉴 → 매장 상세 드로어 내 탭으로 이동
- **랜덤 플리 생성**: 별도 메뉴 → 플레이리스트 관리 탭 내 "랜덤 생성" 모드로 통합
- **음원 데이터 관리**: 탭 방식 → 드롭다운 선택 방식으로 변경, 엑셀 업/다운로드 개선
- **모바일 반응형**: 768px / 480px 기준 전면 개선

## 3. 매장 관리 — 드로어 변경

### 3-1. 탭 추가

| 탭명 | 설명 |
|------|------|
| 기본 정보 | 기존 유지 |
| 매장 무드 | 기존 유지 |
| 추천곡 관리 | 기존 유지 |
| **음원 매핑** (신규) | 시간대별 무드 매핑 설정 |
| **현재 플리** (신규) | 현재 적용 중인 플레이리스트 조회 |

### 3-2. 음원 매핑 탭

- 매장 영업 시간을 파싱하여 **해당 영업 시간대만** 표시 (기존 24시간 전체 표시에서 변경)
- 영업 시간 형식: `"월·금요일 09:00 ~ 22:00"` 파싱 → 유효 시간 배열 생성
- 야간 영업(예: 22:00~03:00) 처리 지원
- 시간대별 무드 선택 후 "매핑 저장" 클릭 → `localStorage('adminMoodSchedule')` 저장

**저장 구조 (`adminMoodSchedule`):**
```json
{
  "매장명A": { "9": "Happy", "10": "Chill", "11": "Happy", ... },
  "매장명B": { "14": "Jazz", ... }
}
```

### 3-3. 현재 플리 탭

- 탭 진입 시 `GET /api/stores/{storeId}/playlists` 호출
- 응답의 `playlist.songs` 배열을 곡 목록으로 표시
- 곡 수 · 총 재생 시간 헤더에 표시
- 데이터는 엑셀 업로드 탭의 `plCurrentSongs` 캐시와 공유

## 4. 플레이리스트 관리 — 랜덤 생성 모드 추가

### 4-1. 모드 전환

기존 엑셀 업로드 전용이었던 플레이리스트 관리 탭에 **"엑셀 업로드 / 랜덤 생성"** 모드 탭이 추가되었습니다.

### 4-2. 랜덤 생성 — 전체 흐름

```
1. 음원 매핑 확인 (드로어 > 음원 매핑 탭에서 사전 설정 필요)
2. 매장 카드에서 개별 "적용 시각" 날짜·시간 설정
   또는 "일괄 적용 시각" 바에서 날짜·시간 → "전체 적용" 클릭
3. "전체 매장 플리 생성" 버튼 클릭
4. 매핑된 모든 시간대에 대해 음원 풀에서 18곡 무작위 선택
5. POST /api/admin/scheduled-playlists 일괄 전송
6. 결과: 매장별 아코디언 카드 (시간대·곡수·총길이·적용 시각 표시)
```

### 4-3. 적용 시각(scheduledAt) 처리 방식

- 엑셀 업로드와 동일하게 **사용자가 지정한 단일 날짜·시간**이 모든 플레이리스트의 `scheduledAt`으로 사용됨
- 개별 시간대별 `scheduledAt`이 아닌, **"언제부터 이 플레이리스트들을 적용할지"** 를 나타내는 단일 시각

**적용 시각 설정 방법:**

| 방법 | 설명 |
|------|------|
| 매장별 개별 설정 | 각 카드의 날짜·시간 입력 (30분 단위 select) |
| 일괄 설정 | 상단 "일괄 적용 시각" 바 → 날짜·시간 설정 → "전체 적용" 클릭 |

### 4-4. 버튼 활성화 조건

- 매핑이 설정된 매장이 1개 이상이면 "전체 매장 플리 생성" 버튼 활성화
- 전체 매장 중 매핑 없는 매장이 있어도 생성 가능 (해당 매장은 건너뜀)

### 4-5. API 요청

```
POST /api/admin/scheduled-playlists
Authorization: Bearer {adminToken}

{
  "schedulePlaylists": [
    {
      "storeName": "매장A",
      "scheduledAt": "2026-06-05T09:00:00.000Z",
      "songs": [
        { "title": "곡제목", "artist": "아티스트", "vid": "유튜브ID", "playTime": 240 },
        ...
      ]
    },
    {
      "storeName": "매장A",
      "scheduledAt": "2026-06-05T09:00:00.000Z",
      "songs": [ ... ]
    },
    ...
  ]
}
```

> `scheduledAt`은 모든 항목 동일 (사용자 지정 시각). `playTime`은 초 단위 정수.  
> `mood`, `hour` 필드는 내부 처리용으로 API 전송 전 제거됨.

### 4-6. 현재 플리 보기 (랜덤 생성 카드 내)

- 각 매장 카드 하단에 "현재 플리 보기" 토글 버튼 추가
- 클릭 시 `GET /api/stores/{storeId}/playlists` 호출 (첫 클릭에만 API 호출, 이후 캐시)
- 곡 목록을 카드 내 인라인으로 표시

## 5. 음원 데이터 관리 변경

### 5-1. 화면 진입 흐름

| 단계 | 조건 | 표시 |
|------|------|------|
| 초기 | 데이터 없음 | 빈 상태 안내 + 업로드 버튼 |
| 초기 | 데이터 있음 | **요약 화면** ("N개 무드 · M곡") |
| 무드 선택 | 드롭다운에서 선택 | 해당 무드 곡 그리드 |

### 5-2. 무드 선택 — 탭 → 드롭다운 변경

- 기존 가로 스크롤 탭 방식 → `<select>` 드롭다운으로 변경
- "전체 보기" 선택 시 요약 화면으로 복귀
- 무드 선택 시 해당 무드 곡 수 표시: `Happy (42곡)`

### 5-3. 엑셀 업로드/다운로드

- 업로드 바는 **항상 최상단에 노출** (요약·편집 화면 모두)
- **엑셀 다운로드** 버튼 신규 추가
  - SheetJS(XLSX 0.20.3) 사용
  - 무드별 시트 생성: 시트명 = 무드명
  - 컬럼: 제목 / 가수 / 재생시간 / YouTube ID
  - 파일명: `음원데이터_YYYY-MM-DD.xlsx`
  - 데이터 없으면 버튼 비활성화

### 5-4. 곡 입력 그리드

무드 선택 후 표시되는 인라인 편집 테이블:

| 컬럼 | 설명 |
|------|------|
| # | 순서 (자동) |
| 제목 | 곡 제목 (필수) |
| 가수 | 아티스트명 |
| 재생시간 | `mm:ss` 또는 초 단위 |
| YouTube ID | 11자리 vid (선택) |
| 삭제 | 행 삭제 버튼 |

- "행 추가" 버튼으로 빈 행 추가
- "저장" 버튼 클릭 → `localStorage('adminMusicData_v2')` 저장

**저장 구조 (`adminMusicData_v2`):**
```json
{
  "Happy": [
    { "title": "곡제목", "artist": "아티스트", "playTime": "3:45", "vid": "dQw4w9WgXcQ" },
    ...
  ],
  "Chill": [ ... ]
}
```

## 6. 모바일 반응형 개선

### 768px 이하

| 영역 | 변경 사항 |
|------|----------|
| 드로어 | 패딩 축소 (`28px` → `20px`), 탭 폰트·간격 조정 |
| 플레이리스트 모드 탭 | 전폭으로 확장, 탭 균등 분할 |
| 엑셀 업로드 일괄 시각 | `flex-wrap` 적용, "전체 적용" 버튼 전폭 |
| 랜덤 생성 일괄 시각 | `flex-wrap` 적용, 시간 select 전폭 |
| 음원 데이터 업로드 바 | 업로드·다운로드 버튼 균등 분할, 힌트 텍스트 하단 배치 |
| 무드 드롭다운 | 전폭 확장 |
| 음원 입력 테이블 | 가로 스크롤 처리 |

### 480px 이하

| 영역 | 변경 사항 |
|------|----------|
| 전체 패딩 | `page-content` 16px → 12px |
| 드로어 | 탭 폰트 12px, 바디 패딩 16px |
| 음원 업로드/다운로드 버튼 | 각각 전폭 (분할 → 단독) |
| 랜덤 생성 카드 | 패딩 12px, 카드명 폰트 13px |
| 패널 제목 | 16px |

## 7. 로컬 스토리지 키 정리

| 키 | 타입 | 설명 |
|----|------|------|
| `adminToken` | String | 어드민 JWT 토큰 |
| `adminMusicData_v2` | JSON Object | 무드별 음원 데이터 `{ moodName: [{title,artist,playTime,vid}] }` |
| `adminMoodSchedule` | JSON Object | 매장별 시간대 무드 매핑 `{ storeName: { "9": "Happy", ... } }` |

## 8. 사용 API 목록

| 메서드 | 엔드포인트 | 용도 |
|--------|-----------|------|
| `GET` | `/api/admin/stores/{id}` | 매장 상세 조회 |
| `GET` | `/api/stores/{id}/playlists` | 현재 적용 중인 플레이리스트 조회 |
| `POST` | `/api/admin/scheduled-playlists` | 플레이리스트 일괄 등록 (엑셀 업로드 / 랜덤 생성 공통) |

> 위 API 외 기존 매장 관리 관련 API(목록 조회, 상태 변경 등)는 변경 없음.
