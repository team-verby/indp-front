> 버전: v1.0
> 작성일: 2026-06-18

본 문서는 크리에이터 수익 환급(정산)의 **단가 산출 근거, 적립·신청 절차, 세금, 측정·어뷰징 기준, 서버 비용·구현 함의**를 정의한다.

## 1. 정산 재원 — 구독료 분해

구독자 1명당 결제액은 **4,400원(부가세 포함)**이며, 아래 비용을 제외한 금액이 서비스 운영 및 크리에이터 환급의 재원이 된다.

| 항목 | 계산 | 금액 |
|------|------|------|
| 결제액 (소비자가, VAT 포함) | — | **4,400원** |
| (−) 부가가치세 VAT | 4,400 × 10 / 110 | −400원 |
| (−) 카드 결제 수수료 (3.4%) | 4,400 × 3.4% | −150원 |
| **= 순정산액 (정산 가능 재원)** | | **≈ 3,850원** |

> 공급가액 기준 4,000원. 카드 수수료에 붙는 부가세는 매입세액공제되어 순부담은 약 136원이나, 보수적으로 150원으로 계상한다.

## 2. 환급 단가 — 역산 근거

환급은 **구독 중인 사용자가 해당 크리에이터의 음악을 실제 청취한 시간**을 기준으로 산정한다. (구독자 수 기준 아님)

**목표 시나리오 (크리에이터 1인 기준):**
약 **2,000명**이 본인 음악을 **하루 8시간**씩 한 달간 청취 → 약 **50만원** 환급

```
월 총 청취시간 = 2,000명 × 8시간 × 30일 = 480,000 청취시간
환급 단가       = 500,000원 ÷ 480,000시간 = 1.0417원 / 청취시간
```

### 확정 단가

| 단위 | 단가 |
|------|------|
| **1 청취시간당** | **1.04원** (정확값 1.0417원) |
| 1분당 | 0.01736원 |
| 1초당 (구현 집계용) | 0.0002894원 |

**재원 대비 비중 검증** (구독자 2,000명 기준)

| 기준 | 월 금액 | 환급 비중 |
|------|---------|----------|
| 총매출 (gross) | 8,800,000원 | 5.7% |
| 순수익 (net) | 7,700,000원 | **6.5%** |

→ 본 정책은 실질적으로 **"순수익의 약 6.5%를 청취시간 비례로 크리에이터에게 환급"**하는 구조다.

## 3. 환급 계산식

```
크리에이터 적립액 = (활성 구독자의 해당 크리에이터 음악 청취시간) × 1.04원
```

- **활성 구독자**(결제 유효 구독)의 청취만 인정한다. 무료·체험·만료·환불 건은 제외.
- 한 구독자의 청취가 여러 크리에이터에 분산되면, 각 크리에이터의 실청취시간만큼 각각 적립된다.

### 청취 규모별 월 적립액 예시 (크리에이터 1인, 본인 음악 기준)

| 본인 음악 청취 규모 | 월 청취시간 | 월 적립액 |
|--------------------|------------|-----------|
| 2,000명 × 8h | 480,000h | 500,000원 |
| 1,000명 × 8h | 240,000h | 250,000원 |
| 500명 × 8h | 120,000h | 125,000원 |
| 100명 × 8h | 24,000h | 25,000원 |
| 50명 × 8h | 12,000h | 12,500원 |

## 4. 적립 및 정산(출금) 절차

정산은 **자동 지급이 아니라 적립 후 신청형**으로 운영한다.

### 4-1. 적립

- 청취시간은 일 단위로 집계되어 크리에이터 **적립금 잔액**에 누적된다.
- 적립금은 소멸되지 않고 계속 이월된다. (휴면·탈퇴 시 별도 처리 — §4-4)

### 4-2. 정산 신청 조건

| 항목 | 기준 |
|------|------|
| **최소 신청 금액** | 적립 잔액 **50,000원 이상** |
| 신청 방식 | DJ 대시보드 정산 카드에서 **크리에이터가 직접 신청** |
| 신청 마감 | 매월 말일 23:59까지 접수분 |
| 지급일 | 익월 **15일** |
| 출금 단위 | 잔액 전액 출금 (또는 5만원 이상 부분 출금) |
| 미신청 / 미달 | 다음 달로 자동 이월 (소멸 없음) |

> 적립 잔액이 5만원 미만이면 신청할 수 없으며, 5만원을 넘길 때까지 계속 적립된다.
> 예: 50명 × 8h 규모(월 12,500원)는 약 4개월 적립 후 신청 가능.

### 4-3. 세금 처리 (지급 시점)

| 구분 | 처리 방식 | 실수령 예시 (신청 50,000원) |
|------|----------|---------------------------|
| 개인 크리에이터 | 정산액의 **3.3% 원천징수** 후 지급 | 50,000 − 1,650 = **48,350원** |
| 사업자 크리에이터 | 세금계산서 역발행 후 지급 (VAT 별도) | 정책에 따름 |

> 연간 수령액이 일정 기준 초과 시 종합소득세 신고 의무가 발생하며, 세무 신고는 크리에이터 본인 책임이다.

### 4-4. 휴면·탈퇴 시

- 휴면 전환 시에도 적립 잔액은 유지된다.
- 탈퇴 시 확정된 적립 잔액은 5만원 미만이라도 마지막 정산일에 지급(원천징수 적용)한다.

## 5. 지속가능성 안전장치

고정 단가(1.04원/h)는 **구독자당 평균 청취시간이 8시간을 크게 초과**하면 환급액이 순수익보다 빠르게 증가해 마진을 잠식할 수 있다.

- **모니터링 기준:** 전체 크리에이터 월 환급 총액이 **순수익의 7%를 초과**하면 단가 재검토 트리거가 작동한다.
- **조정 방식:** 단가 변경은 **시행 30일 전 공지**하며, 이미 적립된 잔액은 **소급 환수하지 않는다.** (변경은 향후 적립분에만 적용)
- 이로써 크리에이터에게는 예측 가능한 단가를, 플랫폼에는 마진 보호를 동시에 보장한다.

## 6. 청취시간 측정 및 어뷰징 방지

환급의 기준값이므로 측정 정의를 엄격히 적용한다.

| 항목 | 기준 |
|------|------|
| 집계 대상 | **실제 재생된 오디오 시간만** (일시정지·정지·탭 비활성 제외) |
| 전송 방식 | 클라이언트가 재생시간을 누적했다가 **5분 단위 또는 일시정지·트랙변경·이탈 시** 델타 전송 |
| 구독 조건 | **활성 구독자**의 청취만 인정 |
| 일일 상한 | 사용자 1인당 **하루 18시간**까지만 집계 (봇·무한루프 방지) |
| 최소 단위 | 30초 미만 스킵 재생은 미집계 |
| 어뷰징 차단 | 동일 IP 다수 접속, 24h 연속 재생, 자동 재생 봇 등은 자동 제외 및 해당 수익 무효 |

> 어뷰징으로 확인된 비정상 청취의 적립분은 즉시 무효 처리되며, 이미 지급된 경우 환수 대상이 된다.

## 7. 서버 비용 영향 (구현 시)

청취시간 추적은 신규 이벤트 로깅이 필요하므로, **저비용 설계**를 전제로 한다.

| 방식 | 일일 쓰기량 (2,000명×8h) | 평가 |
|------|------------------------|------|
| 60초 heartbeat | 약 96만 건/일 | △ 정확하나 쓰기량 많음 |
| **5분(300초) 누적 전송 + 일별 집계 UPSERT** | 약 19만 건/일 (평균 ~2.2req/s) | ◎ 권장 |

- 원본 이벤트는 저장하지 않고 **일별 집계(UPSERT)만** 유지 → 스토리지 거의 0.
- 신규 인프라(서버·DB 인스턴스) 불필요, 기존 EC2/RDS로 흡수 가능 → **증분 비용 미미**.
- heartbeat 주기가 비용 레버이며, 길수록 저렴(정확도와 트레이드오프).

## 8. 구현 함의 (데이터 모델 — 참고)

| 테이블 | 용도 | 주요 컬럼(안) |
|--------|------|--------------|
| `listening_daily` | 일별 청취 집계 | subscriber_id, creator_id, ymd, seconds (UPSERT 누적) |
| `creator_balance` | 크리에이터 적립 잔액 | creator_id, balance, updated_at |
| `creator_ledger` | 적립·출금 원장 | creator_id, type(적립/출금), amount, ref, created_at |
| `payout_request` | 정산 신청 | creator_id, amount, status(신청/승인/지급), tax, requested_at, paid_at |

> 월 마감 시 `listening_daily` 집계 × 단가 → `creator_balance` 가산(원장 기록). 신청 시 잔액 검증(≥5만) → `payout_request` 생성 → 관리자 승인·지급 → 원천징수 차감 후 잔액 반영.

## 9. 요약 카드

```
정산 재원(순정산액)   구독자당 약 3,850원 (4,400 − VAT 400 − 카드 150)
환급 단가             1.04원 / 청취시간 (= 0.01736원/분)
환급 기준             활성 구독자의 크리에이터별 실청취시간
목표 시나리오         2,000명 × 8h × 30일 = 50만원 (1인 기준)
정산 방식             적립 후 신청형 (자동 지급 아님)
최소 신청 금액        50,000원 이상
신청 마감 / 지급일    매월 말일 / 익월 15일
미달·미신청           다음 달로 이월 (소멸 없음)
원천징수              개인 3.3% / 사업자 세금계산서
지속가능성 상한       순수익의 7% 초과 시 단가 재검토 (소급 환수 없음)
일일 청취 집계 상한   1인당 18시간
```
