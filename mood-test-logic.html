4축 부호로 16개 타입 결정

E축 ≥0 → 'E', <0 → 'Q'  /  T축 ≥0 → 'S', <0 → 'W'
R축 ≥0 → 'I', <0 → 'C'  /  M축 ≥0 → 'M', <0 → 'B'
결과: QWCB ~ ESIM 중 1개 (동점 = 0 → 앞쪽 극(Q/W/C/B) 우선)

16개 결과 타입 정의 (4축 MBTI식)

섹션별 측정 목적

SECTION 1 — 공간 분위기 & 감성 (Q1~Q5)
공간이 주고 싶은 감정적 방향성을 분석. Energy(Q↔E)와 Tone(W↔S) 축에 주로 기여. 조용하고 차분한지(Q) vs 활기차고 에너제틱한지(E), 따뜻하고 감성적인지(W) vs 세련되고 쿨한지(S)를 결정.
SECTION 2 — 손님 행동 & 공간 목적 (Q6~Q10)
손님의 행동 패턴과 공간 운영 목적을 분석. Energy(Q↔E)와 Relation(C↔I) 축에 기여. 함께하는 공간(C)인지 개인적인 공간(I)인지, 활동적(E)인지 차분(Q)인지를 세분화.
SECTION 3 — 음악 취향 & 감각 (Q11~Q15)
공간에 맞는 음악의 방향성을 직접 분석. Music(B↔M)과 Tone(W↔S) 축에 기여. 클래식/어쿠스틱 계열(B) vs 인디팝/일렉트로닉 계열(M), 감성적(W) vs 세련된(S) 음악 방향.
SECTION 4 — 공간 스타일 & 브랜드 취향 (Q16~Q20)
인테리어와 브랜드 감성을 분석해 타입을 확정. Tone(W↔S)과 Relation(C↔I) 축에 주로 기여. 앞선 섹션의 방향성을 보강하거나 역전시켜 최종 4글자 타입을 결정.

결과 도출 알고리즘 (의사 코드)

// 1. 초기화 — 4개 축 점수 (음수 가능)
//   scores[0] = Energy   (-Q / +E)
//   scores[1] = Tone     (-W / +S)
//   scores[2] = Relation (-C / +I)
//   scores[3] = Music    (-B / +M)
let scores = [0, 0, 0, 0]

// 2. 각 답변 선택 시 점수 누적
function onAnswerSelect(questionIndex, answerIndex) {
  // 이전 답변 취소
  if (previousAnswer !== null) {
    scores = scores.map((s, i) => s - prevAnswer.scores[i])
  }
  // 새 답변 점수 가산 (음수 포함)
  scores = scores.map((s, i) => s + selectedAnswer.scores[i])
}

// 3. 20문항 완료 후 4축 부호로 타입 결정
function getResult() {
  const e = scores[0] >= 0 ? 'E' : 'Q'
  const t = scores[1] >= 0 ? 'S' : 'W'
  const r = scores[2] >= 0 ? 'I' : 'C'
  const m = scores[3] >= 0 ? 'M' : 'B'
  return e + t + r + m   // 예: 'QWCB', 'ESIM' 등 16가지
}

문항별 최대 기여 점수 히트맵

각 문항에서 선택 가능한 답변 중 각 축에 줄 수 있는 최대 절댓값 점수. 셀이 진할수록 해당 문항이 그 축에 더 큰 영향을 줌. 음수(-) = 앞쪽 극(Q/W/C/B), 양수(+) = 뒤쪽 극(E/S/I/M).

0 ±1 ±2 ±3

문항별 답변 점수 상세

각 문항을 클릭하면 답변별 4개 축 점수 배분을 확인할 수 있습니다. 음수(-) = 앞쪽 극, 양수(+) = 뒤쪽 극.
